Git配置

基础配置

配置用户名 $ git config – global user.name

配置邮箱 $ git config – global user.name

$ git config –global –list 查询到自己设置的用户名和邮箱

这个配置文件会出现在C:\Users\Administrator目录下的 .gitconfig 文件中

配置SSH公钥

在用户文件下创建一个 .ssh 文件，并进入此文件

$ ssh-keygen -t rsa -C “邮箱” -f .ssh/newFileName 创建秘钥

$ list 查看秘钥并使用vi打开.pub文件并复制秘钥到GitHub

创建一个仓库文件夹并使用Git打开

$ git init 创建仓库

建立远程连接

测试上传代码，在需要上传的文件目录下右键然后点击Git Bash Here，第一次提交的时候需要填写远程仓库url和对应远程分支名：

$ git add .git co

$ git commit -m “提交信息”

$ git remote add origin ‘远程仓库url’

$ git push -u origin 对应远程分支名

复制SSH链接

$ git clone git@github.com:savebiu/repo.git git clone 链接

错误提示： error: remote origin already exists.

1、先输入git remote rm origin 删除关联的origin的远程库
2、关联自己的仓库 git remote add origin https://gitee.com/xxxxxx.git
3、最后git push origin master，这样就推送到自己的仓库了

建立Hexo环境

先安装Node.js并在控制台cmd下检测是否安装成功

创建Hexo文件夹并用Git打开

$ npm install -g hexo-cli 安装hexo

$ hexo init 初始化

创建一个Github仓库库名必须是 用户名.github.io

$ npm install 安装组件

$ hexo server 测试

安装 hexo-deployer-git插件：

npm install hexo-deployer-git –save

修改 _config.yml 文件末尾的 Deployment 部分，修改成如下：

deploy:
type: git
repository: git@github.com:用户名/用户名.github.io.git
branch: master

运行 hexo d 将网站上传部署到 GitHub Pages

创建 SSH 密匙

ssh-keygen -t rsa -C “你的 GitHub 邮箱”

粘贴复制的 id_rsa.pub 内容到 Key 中

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。
